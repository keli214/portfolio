Monte Carlo Path Tracer

Implemented a physically based renderer in C++ with Embree that renders virtual 3D scenes with global illumination using Monte Carlo algorithm.

Basic Ray Tracer

This project started as part of the final programming assignment in CSE167: Introduction to Computer Graphics at UCSD, where I implmented a Whitted-Style ray tracer in c++ from scratch. Rather than looping through each pixel for each object in the 3D scene which is used in rasterization, ray tracing generate light paths from the camera center toward the light sources. The basic ray tracer included features like refraction and soft shadow. Below are some images generated from the basic ray tracer:

Monte Carlo Path Tracer

The project was later extended in CSE169: Rendering to generate accurate physically-based rendering with Monte Carlo integration, while the speed performance of the ray tracring was improved by Embree. While the point light source in basic ray tracing generated good-looing results, they are unable to realistic lighting effects. Monte Carlo algorithmm enables global illumination by intergrating over all the illuminace arriving to a single point on the surface of a subject.

Smoothed Normal

To make the coarse facets looks smooth while avoiding increasing the number of facets, we append precomputed per-vertex surface normal as vertex attributes, then smoothly interpolate dN/du and dN/dv on the point of interests to achieve a smoothed look.
